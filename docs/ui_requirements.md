Frontend должен полностью покрывать весь функционал, описанный в документации и OpenAPI-спецификации.

Основные требования к стеку:

* React
* TypeScript
* Vite
* React Router
* TanStack Query (React Query)
* Axios
* Material UI (MUI)
* Zustand для глобального состояния
* React Hook Form для форм

Основные требования к дизайну:

Приложение должно иметь максимально минималистичный и профессиональный интерфейс.

Главные принципы дизайна:

* простота;
* интуитивность;
* отсутствие визуального шума;
* минимум цветов;
* максимум свободного пространства;
* акцент на данных и графиках;
* единый стиль всех страниц.

Не нужно делать перегруженные панели управления, большое количество карточек, градиентов, теней и декоративных элементов.

Краткость и понятность интерфейса являются приоритетом.

Реализовать две темы оформления.

Светлая тема:

* белый фон;
* чёрный текст;
* только чёрные и белые цвета;
* при наведении курсора мыши на какой-либо интерактивный объект он должен динамически заполняться чёрным цветом;

Тёмная тема:

* почти чёрный фон;
* тёмно-синие акценты;
* сине-чёрная палитра;
* хорошая читаемость текста.

Переключение темы должно быть доступно из интерфейса и сохраняться между сессиями.

Структура приложения должна включать следующие страницы:

* Dashboard
* Datasets
* Dataset Details
* Experiments
* Experiment Details
* Training Runs
* Run Details
* Models
* Model Details
* Validation

Навигация должна быть простой и понятной.

Использовать левое боковое меню или компактный верхний navbar.

Не использовать сложные многоуровневые меню.

Все таблицы должны поддерживать:

* сортировку;
* поиск;
* фильтрацию;
* пагинацию.

Для длительных операций необходимо отображать:

* progress bar;
* loading состояния;
* уведомления об ошибках;
* уведомления об успешном выполнении.

Особое внимание уделить странице Validation.

Backend возвращает реальные и предсказанные значения временного ряда.

Нельзя отображать весь временной ряд целиком на одном графике.

Пусть:

* actual — реальные значения;
* predicted — предсказанные значения.

Визуализация должна работать следующим образом:

* разбивать временной ряд на сегменты по 2000 точек;
* внутри каждого сегмента отображать каждую пятую точку;
* использовать окно вида:

start
end = start + 2000

Данные для отображения:

actual[start:end:5]
predicted[start:end:5]

Каждый такой сегмент должен отображаться как отдельная визуализация.

Пользователь должен иметь возможность:

* перейти к следующему сегменту;
* перейти к предыдущему сегменту;
* видеть номер текущего сегмента;
* видеть общее количество сегментов.

По сути это должен быть просмотрщик временных окон.

Не отображать сразу десятки тысяч точек на одном графике.

Для графиков использовать scatter plot.

На графике должны отображаться:

* Actual
* Predicted

Важно:

Вместо длинной прокрутки графика пользователь должен переключаться между окнами данных.

Графики должны выглядеть как статичные снимки отдельных участков временного ряда.

Для реализации графиков используй Recharts или Plotly.

Все API-вызовы должны быть вынесены в отдельный слой сервисов.

Все типы TypeScript должны быть автоматически сгенерированы на основе OpenAPI-схемы либо полностью соответствовать ей.

Сгенерируй production-ready frontend с правильной структурой каталогов, типизацией, роутингом, сервисами API, управлением состоянием, темизацией и всеми страницами.
