# ML Trading Platform

## Описание проекта

ML Trading Platform — веб-приложение для управления полным жизненным циклом ML-моделей прогнозирования фондового рынка.

Система предназначена для работы с нейросетевыми моделями прогнозирования цены закрытия финансовых инструментов на основе исторических данных Московской биржи.

Особенность проекта заключается в том, что модели обучаются одновременно на нескольких тикерах и используют несколько временных рядов в качестве признаков и целевых переменных.

---

# Основные возможности

Пользователь может:

* загружать исторические данные;
* управлять датасетами;
* запускать обучение моделей;
* отслеживать процесс обучения;
* управлять экспериментами;
* регистрировать модели;
* валидировать модели;
* сравнивать результаты различных запусков.

---

# Навигация приложения

## Dashboard

Главная страница системы.

Отображает:

* количество датасетов;
* количество экспериментов;
* количество моделей;
* активные обучения;
* последние эксперименты;
* состояние API.

---

## Datasets

Раздел управления данными.

### Загрузка данных с MOEX

Пользователь задаёт:

* название датасета;
* список тикеров;
* дату начала;
* дату окончания;
* интервал свечей.

После завершения загрузки создаётся новый датасет.

---

### Импорт локальных данных

Пользователь указывает путь к каталогу с parquet-файлами.

---

### Импорт из Google Drive

Пользователь указывает ссылку или ID папки.

---

### Таблица датасетов

Для каждого датасета отображаются:

* название;
* источник;
* тикеры;
* количество строк;
* период данных;
* дата создания.

Доступны действия:

* открыть;
* удалить.

---

## Experiments

Аналог страницы Experiments в MLflow.

Для каждого эксперимента отображаются:

* название;
* описание;
* дата создания;
* количество запусков.

---

### Карточка эксперимента

Отображает:

* общую информацию;
* список запусков;
* лучшие результаты;
* связанные модели.

---

## Training

Раздел запуска обучения.

Пользователь выбирает:

* эксперимент;
* датасет;
* тикеры;
* признаки;
* гиперпараметры;
* вычислительный backend.

Поддерживаются исполнители:

* Local
* Yandex DataSphere

---

### Конфигурация модели

Настраиваемые параметры:

* input_chunk_length
* output_chunk_length
* kernel_size
* num_filters
* dilation_base
* num_layers
* learning_rate
* batch_size
* n_epochs
* loss
* device

Также возможна загрузка JSON-конфигурации.

---

### Мониторинг обучения

Во время выполнения отображаются:

* статус;
* номер эпохи;
* количество эпох;
* train loss;
* validation loss;
* итоговые метрики.

Обновление данных должно выполняться автоматически.

---

### График обучения

Строится по истории обучения.

Оси:

X = Epoch

Y = Loss

Линии:

* Train Loss
* Validation Loss

---

## Models

Раздел управления моделями.

Поддерживаются два способа регистрации модели:

### Загрузка файла

Пользователь загружает .pt файл через браузер.

### Регистрация по пути

Пользователь указывает путь к существующей модели.

---

### Таблица моделей

Отображаются:

* название;
* архитектура;
* дата создания;
* связанные тикеры;
* метрики;
* связанный запуск обучения.

---

### Карточка модели

Отображает:

* информацию о модели;
* параметры обучения;
* метрики;
* использованные тикеры;
* связанные эксперименты.

---

## Validation

Раздел проверки качества модели.

Пользователь выбирает:

* модель;
* датасет;
* тикеры;
* признаки.

После запуска отображаются результаты валидации.

---

### Метрики

Отображаются все метрики, возвращённые backend.

Например:

* MAE
* RMSE
* MAPE
* R²

---

### График прогнозов

Для каждого тикера строится scatter plot:

* Actual
* Predicted

---

# Основной пользовательский сценарий

1. Создать датасет.
2. Создать эксперимент.
3. Запустить обучение.
4. Следить за процессом обучения.
5. Получить обученную модель.
6. Зарегистрировать модель.
7. Запустить валидацию.
8. Просмотреть метрики.
9. Сравнить результаты различных запусков.
